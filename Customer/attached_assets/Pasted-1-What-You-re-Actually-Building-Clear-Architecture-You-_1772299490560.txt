2️⃣ Since You’re Using Replit — Use This Structure

On Replit, don’t create complicated monorepos.

Do this instead:

Create 3 Repls:

1️⃣ backend-api
2️⃣ citizen-app
3️⃣ doctor-app

This avoids dependency chaos.

3️⃣ Backend API (Core Brain) — FIRST Build This

Everything depends on this.

Repl #1: backend-api

Stack:
Node.js + Express
SQLite (file DB)
No Prisma needed in Replit (too heavy)

Install:

npm init -y
npm install express cors uuid better-sqlite3

Why SQLite file DB?
Because Replit keeps files persistent.
No cloud DB headache.
Perfect for hackathon.

Backend Features To Implement (In Order)

Build in this exact order:

Step 1 — Aadhaar Simulation

Create:
POST /api/aadhaar/verify

Use simple JSON file:
aadhaar.json

Check:
aadhaarId
name
dob

Return verified true/false.

Done.

Step 2 — Centers Data

Create:
GET /api/centers

Hardcode 3–5 Mumbai centers.

Return:
lat
lng
slots
vaccines available

Done.

Step 3 — Appointment Booking

POST /api/appointments

Store in SQLite:
appointmentId
aadhaarId
name
centerId
slotTime
vaccine
status

Then:

GET /api/appointments?centerId=CEN-001

Doctor app will use this.

Done.

Step 4 — Inventory System

Create:
GET /api/inventory?centerId=
POST /api/inventory/reorder

When reorder happens:
Create shipment record.

Done.

Step 5 — Shipment Simulation

Create:

POST /api/sim/start?shipmentId=

Use setInterval() to:
update checkpoint every 2 seconds.

Doctor app polls:
GET /api/shipments/:id

That’s enough.
No WebSockets needed in Replit.

Step 6 — QR Scan Stub

POST /api/qr/scan

Just accept:
qrPayload

Return:
ok true
shipmentId parsed

Done.

Step 7 — Voice Stub

In hackathon, do NOT integrate heavy STT models locally.

Use:

Browser SpeechRecognition API for speech → text
Browser SpeechSynthesis for text → speech

Much easier.

So backend voice endpoints can be skipped for now.
