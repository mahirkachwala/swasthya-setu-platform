5. TELEMETRY API

Create endpoint:

POST /api/iot/telemetry

Payload:
{
shipmentId,
temperature,
humidity,
pressure,
latitude,
longitude,
tamperFlag,
timestamp
}

Store raw telemetry in MongoDB.

6. RULE ENGINE (CRITICAL LOGIC)

When telemetry arrives:

Check temperature range.

Check tamper flag.

Check geofence deviation.

If violation detected:

Create violation event

Generate canonical JSON

Hash event using SHA256

Store event locally

Call blockchain service

7. BLOCKCHAIN LOGIC

Smart Contract Requirements:

recordViolation(shipmentId, eventHash, eventType, timestamp)

Emit event:
ViolationRecorded(shipmentId, eventHash, eventType, timestamp)

Do NOT store large data on chain.

Only store:

shipmentId

eventHash

eventType

timestamp

Backend must:

Use ethers.js

Sign transaction

Wait for confirmation

Store txHash in MongoDB

8. QR CODE LOGIC

Each vaccine vial has QR code containing:

{
shipmentId,
latestStatus,
blockchainTxHash,
verificationURL
}

When doctor scans:

GET /api/vaccine/:shipmentId

Return:

status SAFE / UNSAFE

violation history

blockchain proof

last telemetry
