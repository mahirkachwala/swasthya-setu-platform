2️⃣ MAIN DASHBOARD (Overview Screen)

This is the homepage after login.

Purpose

Give doctor an instant snapshot of:

Today's appointment count

Total vaccines available

Incoming shipments

Capacity utilization

Alerts

UI Layout

Top section: KPI cards

Metric	Meaning
Today’s Appointments	Total booked today
Checked-in Patients	Status = CHECKED_IN
Vaccinated	Status = VACCINATED
Available Stock	Total vaccine units
Incoming Shipments	Status = CREATED or IN_TRANSIT
API Used
Get today’s appointments
GET /api/appointments?centerId=CEN-001&date=YYYY-MM-DD
Get inventory
GET /api/inventory?centerId=CEN-001
Get shipments
GET /api/shipments?centerId=CEN-001
Behind the Scenes

Doctor dashboard calculates:

Capacity Used = total appointments / openSlots
Low Stock Alert = any vaccine < 20 units
Shipment Arriving Alert = shipment.status == ARRIVED

4️⃣ INVENTORY MANAGEMENT
Purpose

Track vaccine stock levels.

Doctor must know:

Which vaccines are available

How many doses left

Whether to reorder

UI Layout

Table format:

Vaccine Type	Quantity	Status
Covishield	120	OK
Covaxin	15	Low Stock
Logic

If quantity < 20 → show red badge “LOW STOCK”

Reorder Section

Doctor selects:

Manufacturer

Vaccine Type

Quantity

POST request:

POST /api/inventory/reorder
{
  centerId: "CEN-001",
  manufacturer: "Serum Institute",
  vaccineType: "Covishield",
  quantity: 500
}

Response:

{
  shipmentId: "SHIP-12345",
  status: "CREATED"
}

This automatically creates a shipment record.

5️⃣ SHIPMENT TRACKING PAGE

This is the most visually impressive part.

Purpose

Simulate vaccine transport from manufacturer to clinic.

UI Sections
1. Shipment List
Shipment ID	Vaccine	Qty	Status	Action
SHIP-12345	Covishield	500	CREATED	Start
2. Map View

Show origin

Show checkpoints

Show current position marker

Draw polyline path

API Calls
Fetch shipment list
GET /api/shipments?centerId=CEN-001
Fetch shipment detail
GET /api/shipments/SHIP-12345
Start simulation
POST /api/sim/start?shipmentId=SHIP-12345
What Happens When Simulation Starts

Backend timer:

Every 2 seconds:

Move to next checkpoint

Update currentLat

Update currentLng

Update currentCheckpoint

Doctor UI polls every 2 seconds:

GET /api/shipments/SHIP-12345

When final checkpoint reached:
status = ARRIVED

When ARRIVED

Dashboard should show:

🟢 Shipment Arrived
🔔 Inventory ready for scanning

6️⃣ QR SCAN PAGE
Purpose

Simulate scanning incoming vaccine vials.

Important:
This does NOT validate blockchain.
It only sends payload to backend stub.

UI

Option A (fastest):
Input box:

Paste QR payload here

Example payload:

SHIP-12345|BATCH-2026-009|sig-demo

Button: Scan

API
POST /api/qr/scan
{
  "qrPayload": "SHIP-12345|BATCH-2026-009|sig-demo"
}

Response:

{
  ok: true,
  shipmentId: "SHIP-12345",
  message: "Accepted (demo). Blockchain validation handled separately."
}
